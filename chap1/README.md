# Test

## 테스트를 하는 이유

1. 기능 정상 동작
2. 요구 사항 만족
3. 이슈 예측
4. 버그 발견
5. 리팩토링
6. 유지보수
7. 의존성 감소
8. 문서화

> 자신감을 가질 수 있다!

---

## 테스트 종류

### 1. Unit Test

- 단위 테스트
- 속도가 빠르다.

### 2. Integration Test

- 통합 테스트

### 3. E2E Test

- UI 테스트, 사용자 테스트
- 비용이 크다.

---

## TDD

1. 테스트 작성
2. 테스트 실패
3. 테스트를 통과하기 위한 최소한의 코드 작성
4. 테스트 성공
5. 리팩토링

> 요구사항을 이해하고 사용자 입장에서 코드를 작성할 수 있다.

---

## TDD를 할 때

1. 요구사항이 명확할 때
2. 비지니스 로직일 때
3. 협업을 위한 명세서(문서)가 필요할 때
4. 설계에 대한 고민이 필요할 때

> UI는 따로 TDD로 작성하지 않고, 다 만든 뒤에 확인만 하는 경우가 많다.

---

## CI/CD에서의 TDD

- CI 단게에서 테스트가 자동화 되도록 해야한다.
